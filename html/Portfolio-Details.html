졸업작품 목적으로 시도하였던 여행경비 예측 사이트를 국비지원 과정에서 EC2, RDS를 통해 배포까지 성공한 사이트입니다.
데이터는 한국관광데이터랩의 데이터를 사용하였으며 데이터가 5년치로 한정되어 있어 비교적 정확도는 부족합니다.
예측 자체는 파이썬으로 하였으며, 사이트는 Spring 으로 제작하였습니다.

문제점
프로젝트 진행 중, 크게 두가지 문제점을 겪었습니다.

1. 개발에 필요한 실질적인 능력 부족
• 프로젝트 처음 진행 당시 개발에 대한 아무런 지식이 없었기에 예측까지는 성공하였으나
사이트를 완성하지 못했습니다.
• 프론트엔드는 만들 수 있었으나 백엔드에 대한 개념이 전무하였기에 프론트엔드와 백엔드를
연결시키지 못해 사이트가 제 기능을 하지 못하였습니다.
• 때문에 졸업작품 발표 당시에는 사이트와 예측치를 각각 별도로 발표하였습니다.

2. Docker, EC2 및 RDS에 대한 개념 부재
• Docker에 대한 개념이 전무하여 이미지 빌드를 시도하지도 않고 바로 EC2를
사용하려고 하였습니다.
• EC2와 RDS를 연결시키지 않아 배포에는 성공하였음에도 불구하고 서비스를 이용 시
오류가 발생하였습니다.
해결법
1. 백엔드 개발에 필요한 지식 습득: 국비지원 과정을 거치며 Spring 프레임워크와 백엔드 개발에
필요한 실질적인 지식을 쌓았습니다. 이를 통해 국비지원 과정 중 졸업작품을 재 진행하여 배포까지
진행하였습니다.

2. EC2와 RDS 연결: EC2에 대해 처음 Docker 컨테이너를 업로드 후, EC2를 RDS와 연결시켜
서비스가 제대로 작동하게 하였습니다. 그 결과 서비스가 제대로 작동하는 것을 확인하였습니다.

배운 내용
GeoJson을 통한 지도 시각화, 파이썬을 통한 비율 계산 및 분석
배운 내용
1. GeoJson 사용방법
• GeoJson 시각화 시에 특정 모양으로 시각화 하려면 Json 파일이 필요하다는 것을 배웠습니다.
• 지도 시각화와 범례 사용에 대해 자세히 배울 수 있었습니다. 지도 시각화는 몇 배율로 표시하는지가 중요한데 이를 범례를 통해 더욱 시각적으로 표시하는 방법을 배웠습니다.

2. 데이터 전처리 및 결합
• 데이터를 전처리 시에 pandas를 통해 더욱 효율적으로 전처리하는 방법을 배웠습니다.

3. 객관적인 분석의 중요성
• 이 대회를 진행할 당시 단순한 분석 및 시각화로 그쳤으며, 이를 팀원들과 함께 임의로
분석하였습니다. 이로 인해 좋지 못한 성적을 거두었으며, 이를 계기로 데이터를 분석할 때,
더욱 객관적인 과정 및 결과가 필요하다는 것을 크게 배웠습니다.
문제점
1. 데이터 키(지역 코드) 불일치 문제
• 지도 시각화에 익숙치 않아 데이터키와 지역을 제대로 매칭시키지 못하였고 이로 인해 특정 지역에 대한 시각화가 제대로 진행되지 않는 문제가 발생하였습니다.

2. 지역 누락 문제
• Json 파일을 처음부터 만드는 것은 불가능하다고 판단하여 온라인에서 기존의 파일을 찾아
수정하여 사용하였습니다.
그러다보니 제가 분석한 지역과는 매칭이 되지 않았고, 이로 인해 특정 지역에 대한 시각화가
진행되지 않는 문제가 발생하였습니다.
해결법
1. 데이터 키 수정: 엑셀의 데이터 키를 Json 파일에 맞게 수정하여 데이터키 불일치 문제를
해결하였습니다.

2. 새로운 Json 파일 탐색: 온라인에서 전국 8도로 나누어진 Json 파일을 찾아 새로운 Json 파일로 설정, 지역 누락 문제를 해결하였습니다. 물론 Json 파일 없이 코드 자체에서 해결할 수 있었지만
저희는 전국 8도라는 특정한 모양의 지도 시각화를 원하였고, Json 파일을 찾는 것이 더욱
효율적이라는 판단하에 시행하였습나다.
